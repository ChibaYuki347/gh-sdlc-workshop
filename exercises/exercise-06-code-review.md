# 演習6: Copilot Code Review と品質ゲート演習

**対応セッション**: セッション5 - PR を中心とした品質ゲート
**所要時間**: 約30分

---

## 🎯 演習のゴール

- Copilot Code Review を PR 上で体験する
- AIレビュー + 人間レビューの組み合わせを理解する
- PR が品質ゲートとして機能する流れを体験する

---

## 📝 演習手順

### Step 1: Pull Request を作成する（10分）

#### 変更をコミット・プッシュ

```bash
# すべての変更をステージング
git add .

# コミットメッセージを作成（Copilotに提案してもらう）
git commit -m "feat: 契約更新通知機能の追加とレガシーコードのリファクタリング

- 契約更新通知サービスの新規実装
- 保険料計算ロジックの分離とリファクタリング
- PolicyStatus enum によるステータス管理の改善
- ユニットテストの追加"

# プッシュ
git push origin feature/team-X-renewal-notification
```

#### PR説明文を Copilot で生成

VS Code Copilot Chat で:

```
このブランチの変更内容に基づいて、Pull Requestの説明文を日本語で作成してください。

以下のセクションを含めてください:
## 概要
## 変更内容
## テスト結果
## レビューポイント（レビュアーに特に見てほしい点）
## チェックリスト
```

GitHub で PR を作成し、生成された説明文を使用してください。

### Step 2: Copilot Code Review を実行（10分）

#### Copilot をレビュアーに追加

1. PR 画面の **Reviewers** セクションを開く
2. **Copilot** を Reviewer として追加
3. Copilot が自動でコード分析を開始（数分待つ）

#### レビューコメントを確認

Copilot Code Review のコメントを確認:

```
確認ポイント:
□ バグの可能性の指摘
□ セキュリティリスクの指摘
□ コード品質の改善提案
□ パフォーマンスの懸念
```

#### 指摘への対応

重要な指摘に対して修正を行い、再プッシュ:

```bash
# Copilot Chat で指摘対応を依頼
# 「Copilot Code Review で以下の指摘がありました。修正してください: (指摘内容)」

git add .
git commit -m "fix: Copilot Code Review 指摘への対応"
git push
```

### Step 3: 人間によるレビュー + マージ（10分）

**他チームのPRをレビューする**

講師がチーム間のPR割り当てを指示します。

#### レビュー観点

```
□ コードの可読性
  - 変数名・メソッド名は意味が分かるか
  - コメントは適切か

□ 仕様との整合性
  - 仕様書の要件を満たしているか
  - 受け入れ条件がテストで確認されているか

□ 設計・構造
  - 適切な責務分離がされているか
  - 既存コードとの一貫性があるか

□ エラーハンドリング
  - null チェックは適切か
  - 例外処理は適切か

□ テスト
  - テストカバレッジは十分か
  - エッジケースがテストされているか
```

#### Copilot Code Review + 人間レビューの併用

Copilot Code Review は **機械的なチェック**（バグ、セキュリティ、パターン）を自動化します。
人間のレビューは **ビジネスロジックの正しさ**、**設計判断**、**チーム規約** に集中できます。

```
Copilot Code Review の担当:
  ✅ null チェック漏れ
  ✅ セキュリティリスク
  ✅ パフォーマンス懸念
  ✅ コードパターンの改善

人間のレビューの担当:
  ✅ 仕様との整合性
  ✅ ビジネスロジックの正しさ
  ✅ 設計方針との一貫性
  ✅ チーム規約への準拠
```

#### 全チェック通過後にマージ

```
PR の品質ゲート:
  ✅ Copilot Code Review  → 対応済み
  ✅ CI / テスト          → パス
  ✅ 人間のレビュー        → 承認
  → マージ可能！
```

---

## ✅ 完了条件

- [ ] Copilot Code Review のコメントが PR に付いている
- [ ] Copilot の指摘に対して修正を実施済み
- [ ] 他チームの PR に人間レビューコメントがある
- [ ] 品質ゲート（Review + CI）を通過してマージ済み（or マージ可能状態）
