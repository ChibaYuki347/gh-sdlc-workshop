# Insurance CRM Workshop Constitution

## Core Principles

### I. レガシーコードの段階的改善
既存コードにはレガシーパターン（God Class、マジックナンバー等）が含まれる。
リファクタリングは段階的に行い、既存APIの互換性を常に維持すること。

### II. 仕様駆動開発（SDD）
すべての機能追加・変更は仕様書（spec.md）から始める。
仕様 → 計画 → タスク → 実装 の流れを遵守する。

### III. テストファースト
新規実装にはテストコードを伴わせる。
JUnit 5 + Spring Boot Test を使用する。

### IV. 日本語ドキュメント
仕様書・コメント・コミットメッセージは日本語を基本とする。
コード（クラス名・メソッド名・変数名）は英語で記述する。

### V. REST API 規約
エンドポイントは `/api/` プレフィックス、リソース名は複数形。
適切な HTTP ステータスコードとエラーレスポンスを返す。
