## 認証トラブルシューティング

### git push が拒否される場合

```bash
# 方法1: ブラウザ認証（Git Credential Manager）
git push origin feature/team-X-setup
# → ブラウザが開くのでリポジトリ用アカウントでログイン

# 方法2: Personal Access Token (PAT) を使用
git remote set-url origin https://<PAT>@github.com/<org>/<repo>.git

# 方法3: SSH キー（事前設定が必要）
git remote set-url origin git@github.com:<org>/<repo>.git
```

### VS Code で Copilot が切れた場合

```
1. VS Code 左下のアカウントアイコンをクリック
2. "Sign in with GitHub to use GitHub Copilot" を選択
3. Copilot 用アカウント（firstname-lastname_zurichcb）でサインイン
4. Copilot アイコンが有効に戻ることを確認
```

> ⚠️ **注意**: ブラウザの GitHub セッションを切り替えると、
> VS Code の認証に影響する場合があります。
> ブラウザでリポジトリ用アカウントにログインした後、
> VS Code の Copilot が無効になっていないか確認してください。
