# 演習5: テスト生成演習

**対応セッション**: セッション5 - テスト生成とセキュアなコードレビュー
**所要時間**: 約30分

---

## 🎯 演習のゴール

- Copilot を使ってユニットテストを自動生成する
- テストの品質を評価し、不足をCopilotに指摘して改善する
- テスト駆動のリファクタリングの安全性を体感する

---

## 📝 演習手順

### Step 1: 既存コードのテスト生成（10分）

`CustomerService.java` の `calculatePremium` メソッドに対するテストを生成します。

VS Code で `CustomerService.java` を開き、`calculatePremium` メソッドを選択した状態で Copilot Chat に以下を入力:

```
選択したメソッド calculatePremium に対する JUnit 5 のユニットテストを作成してください。

テストクラスの場所:
app/src/test/java/com/zurich/crm/service/CustomerServiceTest.java

以下のテストケースを含めてください:

【正常系】
- 生命保険(LIFE): 30歳以下、非喫煙者 → 基本料金 × 1.0 × 税
- 生命保険(LIFE): 45歳、喫煙者 → 基本料金 × 1.8 × 1.5 × 税
- 医療保険(MEDICAL): 65歳、非喫煙者 → 基本料金 × 3.0 × 税
- 自動車保険(AUTO): 20歳 → 基本料金 × 2.0 × 税
- 火災保険(FIRE): 基本料金 × 税

【境界値】
- 年齢 30歳ちょうど（LIFE の料率変更境界）
- 年齢 40歳ちょうど
- 年齢 60歳ちょうど
- 年齢 25歳ちょうど（AUTO の料率変更境界）

【異常系】
- 存在しないポリシータイプ
- null のポリシータイプ

Spring Boot Test + JUnit 5 + Mockito を使用してください。
```

### Step 2: テストを実行する（5分）

```bash
cd app
mvn test
```

**テストが失敗した場合**:

エラーメッセージを Copilot Chat に貼り付けて修正を依頼:

```
以下のテストが失敗しました。原因を分析して修正してください:

（エラーメッセージを貼り付け）
```

### Step 3: リファクタリング後のテスト生成（10分）

演習4で作成した `PremiumCalculatorService` に対するテストを生成:

```
PremiumCalculatorService のユニットテストを作成してください。

テストクラスの場所:
app/src/test/java/com/zurich/crm/service/PremiumCalculatorServiceTest.java

以下を含めてください:
1. 各保険タイプの基本計算テスト
2. リファクタリング前の CustomerService.calculatePremium と
   同じ入力に対して同じ出力が得られることを確認する回帰テスト
3. 入力バリデーションのテスト（不正な保険タイプ、不正な年齢）
4. エッジケースのテスト
```

### Step 4: テストカバレッジの確認（5分）

```
作成したテストのカバレッジを評価してください。
テストされていないコードパスはありますか？
追加すべきテストケースがあれば提案してください。
```

> 💡 Copilot に「足りないテスト」を聞くと、見落としていたエッジケースが見つかることがあります

## 💡 Tips

### よくあるテスト生成時の問題と対処

| 問題 | 対処 |
|------|------|
| import が足りない | 「必要な import 文も含めてください」と追加依頼 |
| Mockの設定が不正確 | テスト対象クラスの依存関係を明示して再生成 |
| 期待値が合わない | 実際の計算結果を確認し、テストの期待値を修正 |
| テストが実装に依存しすぎ | 「振る舞いベースのテストにしてください」と依頼 |
