# セッション5: テスト生成とセキュアなコードレビュー

**時間**: 14:45 - 16:00（75分）

---

## 🎯 セッションのゴール

- Copilot を使ったテストコードの自動生成を体験する
- GitHub Advanced Security の概念を理解し、セキュリティを意識した開発を学ぶ
- Copilot を活用した Pull Request の作成とコードレビューを体験する

---

## 📖 アジェンダ

### 1. テストコードの自動生成（30分）

#### なぜテストが重要か

- レガシーコードの改修時、**既存動作を壊さない**ことの保証
- リファクタリングの安全ネット
- 仕様書（期待動作）の実行可能なドキュメント

#### Step 1: 既存コードのテスト生成

`CustomerService.java` の `calculatePremium` メソッドに対するテストを生成:

```
プロンプト:
「CustomerService.java の calculatePremium メソッドに対する
JUnit 5 のユニットテストを作成してください。

以下のテストケースを含めてください:
- 各保険タイプ（LIFE, MEDICAL, AUTO, FIRE）の基本保険料
- 年齢による料率変動
- 喫煙者の割増
- 境界値テスト（年齢 25, 30, 40, 60, 65）
- 無効な保険タイプの入力」
```

#### Step 2: リファクタリング後のテスト

セッション4でリファクタリングした `PremiumCalculatorService` に対するテスト:

```
プロンプト:
「PremiumCalculatorService のユニットテストを作成してください。
リファクタリング前と同じ計算結果になることを確認する回帰テストも含めてください。」
```

#### Step 3: テストの実行

```bash
cd app
mvn test
```

テストが通らない場合:
```
プロンプト:
「以下のテストエラーを修正してください:
（エラーメッセージを貼り付け）」
```

### 2. セキュリティを意識した開発（15分）

#### GitHub Advanced Security の概要

| 機能 | 説明 |
|------|------|
| **Code Scanning** | コード内のセキュリティ脆弱性を自動検出 |
| **Secret Scanning** | パスワード、APIキーなどの秘密情報の混入を検出 |
| **Dependabot** | 依存ライブラリの脆弱性を検出・自動更新 |
| **Supply Chain Security** | サプライチェーン攻撃への防御 |

#### Copilot でセキュリティレビュー

```
プロンプト:
「CustomerController.java のセキュリティ上の問題点を指摘してください。
OWASP Top 10 の観点から分析してください。」
```

**発見すべき脆弱性の例**:
- 入力バリデーションの欠如（インジェクション攻撃のリスク）
- エラーハンドリング不足（情報漏洩のリスク）
- 認証・認可の欠如
- レート制限の不在

#### Copilot でセキュリティ修正

```
プロンプト:
「指摘されたセキュリティ問題を修正してください:
1. @Valid アノテーションによる入力バリデーション追加
2. グローバル例外ハンドラの追加（内部エラー情報を隠蔽）
3. APIレート制限の検討」
```

### 3. Pull Request とコードレビュー（30分）

#### Step 1: 変更をコミットしてPRを作成

```bash
# 変更をステージング
git add .

# コミット（Copilotにコミットメッセージを提案してもらう）
git commit -m "feat: 契約更新通知機能の追加とレガシーコードのリファクタリング"

# プッシュ
git push origin feature/team-X-renewal-notification
```

GitHub で Pull Request を作成:
- **タイトル**: わかりやすい変更概要
- **説明**: Copilot Chat に PR の説明文を生成してもらう

```
プロンプト:
「以下の変更内容に基づいて、Pull Requestの説明文を作成してください:
- 契約更新通知機能の新規追加
- 保険料計算ロジックのリファクタリング
- ユニットテストの追加
- セキュリティ修正

日本語で、変更の目的・内容・テスト結果を含めてください。」
```

#### Step 2: コードレビューの実施

**他チームのPRをレビューする**（チーム間でPRを交換）

レビューのポイント:
```
□ 仕様書の要件を満たしているか
□ コードの可読性は十分か
□ テストは適切にカバーされているか
□ セキュリティ上の問題はないか
□ パフォーマンスの懸念はないか
```

#### Copilot を活用したレビュー

PR のコードを選択して:
```
プロンプト:
「このコードをレビューしてください。
改善点、バグの可能性、設計上の懸念を指摘してください。」
```

#### Step 3: レビューコメントの対応

レビューで指摘された内容を Copilot の支援で修正し、再度コミット。

## 💡 講師向けメモ

- テスト生成は参加者の経験レベルに合わせて難易度調整
- セキュリティの話題はビジネスメンバーにも関心が高い。具体例を多めに
- PRレビューはチーム間で交換すると学びが大きい
- 時間が足りない場合はセキュリティセクションを簡略化し、テストとPRに集中
