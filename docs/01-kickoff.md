### 1. ワークショップの目的と進め方（10分）

**本日のテーマ**: ソフトウェア開発のあり方の変革（モダナイゼーション）

本ワークショップでは、単なるツールの使い方を学ぶのではなく、**開発ライフサイクル全体がどう変わるのか**を体験します。

```
従来の開発フロー:
  要件定義 → 設計 → 実装 → テスト → レビュー → デプロイ
  （各フェーズが分断、手戻りが多い）

AI時代の開発フロー:
  仕様策定 ⇄ AI実装 ⇄ テスト ⇄ レビュー
  （AIがフェーズ間の橋渡し、イテレーションが高速化）
```

**本日のハンズオンシナリオ**:
- 擬似保険CRMアプリケーション（Java/Spring Boot）を題材に使用
- レガシーコードの理解 → リファクタリング → 新機能追加 の流れを体験

### 2. GitHub Enterprise の価値（15分）

#### セキュアなコラボレーション基盤

| 機能 | 概要 |
|------|------|
| **リポジトリ管理** | コードの一元管理、バージョン管理、アクセス制御 |
| **Issues & Projects** | 要件・タスク管理をコードと統合 |
| **Pull Request** | コードレビューの標準化、品質向上 |
| **GitHub Actions** | CI/CDパイプラインの自動化 |
| **Advanced Security** | 脆弱性検出、シークレットスキャン、依存関係監査 |
| **Audit Log** | 操作履歴の追跡、コンプライアンス対応 |

#### なぜGitHub Enterprise か

- **Inner Source**: 社内でのオープンソース的な開発文化の醸成
- **統合プラットフォーム**: 要件からデプロイまで一つのプラットフォームで完結
- **エンタープライズセキュリティ**: SAMLシングルサインオン、IP制限、監査ログ

### 3. GitHub Copilot による開発パラダイムシフト（15分）

#### Copilot の進化

```
Phase 1: コード補完（Copilot）
  └ 1行〜数行のコード提案

Phase 2: 対話型AI（Copilot Chat）
  └ コードの説明、デバッグ支援、設計相談

Phase 3: AIエージェント（Copilot Agent Mode）
  └ 仕様からコード生成、テスト生成、リファクタリング
  └ 複数ファイルの横断的な変更
  └ ターミナルコマンドの実行
```

#### 仕様駆動開発（Spec-Driven Development）とは

GitHub Spec Kit を使った SDD は 4 つのフェーズで構成されます:

```
1. Specify（/speckit.specify）
   仕様書を作成 — What/Why を定義
   ↓ AIが仕様を構造化
2. Plan（/speckit.plan）
   技術計画を立案 — How を決定
   ↓ AIが設計を提案
3. Tasks（/speckit.tasks）
   実装タスクに分解 — 具体的な作業単位
   ↓ タスクを GitHub Issues に登録
4. Implement（Copilot Agent）
   タスクに基づいてコードを生成
   ↓ 人間がレビュー・マージ
```

**ポイント**: 「何を作るか（What）」を人間が定義し、「どう作るか（How）」をAIが支援する。
仕様・計画・タスクはすべてリポジトリ内（`.specify/`）で管理され、変更履歴が追跡可能。
