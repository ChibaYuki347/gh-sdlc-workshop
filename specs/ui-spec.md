## 3. 各画面仕様

### 3.1 ダッシュボード（`/`）

**目的**: システムの全体状況を俯瞰し、主要な操作へ素早くアクセスするためのトップページ

**主な表示要素**:
- **サマリー統計パネル**（4列）:
  - 総顧客数（青パネル）
  - 有効契約数（緑パネル）
  - 失効契約数（赤パネル）
  - 月間保険料合計（水色パネル、カンマ区切り円表示）
- **クイック操作パネル**:
  - 「新規顧客登録」ボタン（青・大サイズ） → `/customers/new`
  - 「保険料シミュレーション」ボタン（水色・大サイズ） → `/premium-calculator`
  - 「月次レポート表示」ボタン（黄色・大サイズ） → `/report`
- **最近の顧客テーブル**（最大5件表示）:
  - 列: 氏名（詳細リンク付き）、契約種別、ステータス（ラベル表示）
  - データがない場合は「顧客データがありません」と表示

**ユーザー操作**:
- クイック操作ボタンをクリックして各機能へ遷移
- 最近の顧客の氏名をクリックして顧客詳細画面へ遷移

### 3.2 顧客一覧（`/customers`）

**目的**: 登録済みの全顧客を一覧表示し、検索・各種操作を行う

**主な表示要素**:
- **検索バー**:
  - テキスト入力欄（プレースホルダー: 「氏名・メール・契約番号」、幅300px）
  - 「検索」ボタン（青）
  - 「クリア」ボタン（グレー） → 検索条件をリセット
  - 「+ 新規登録」ボタン（緑、右寄せ） → `/customers/new`
- **フラッシュメッセージ**: 登録・更新・削除後の成功メッセージ（緑アラート）
- **検索結果表示**: 検索時に「"キーワード" の検索結果: N件」を表示（青アラート）
- **顧客テーブル**:
  - 列: ID、氏名（詳細リンク付き）、メール、電話番号、契約番号、契約種別（日本語表示）、ステータス（色付きラベル）、保険料（カンマ区切り円表示）、操作
  - 操作列: 「詳細」ボタン（水色・小）、「編集」ボタン（黄色・小）
  - データがない場合は「顧客データがありません」と表示
- **件数表示**: テーブルヘッダーに「顧客リスト（全N件）」

**ユーザー操作**:
- キーワードを入力して検索（姓、名、メール、契約番号で部分一致検索）
- 「クリア」で全件表示に戻る
- 氏名リンクまたは「詳細」ボタンで顧客詳細画面へ遷移
- 「編集」ボタンで顧客編集画面へ遷移
- 「+ 新規登録」ボタンで顧客登録画面へ遷移

**クエリパラメータ**:

| パラメータ | 型     | 必須 | 説明       |
|-----------|--------|------|-----------|
| keyword   | String | —    | 検索キーワード |

### 3.3 顧客詳細（`/customers/{id}`）

**目的**: 個別の顧客情報と契約内容を詳しく確認する

**主な表示要素**:
- **フラッシュメッセージ**: 登録・更新後の成功メッセージ（緑アラート）
- **基本情報パネル**（左列）:
  - ID、氏名、メール、電話番号、住所、登録日、更新日
- **契約情報パネル**（右列）:
  - 契約番号、契約種別（日本語表示）、ステータス（色付きラベル）、保険料（円/月）、契約開始日、契約終了日
- **担当者情報パネル**:
  - 担当者名、担当者メール
- **操作ボタンエリア**:
  - 「編集」ボタン（黄色） → `/customers/{id}/edit`
  - 「削除」ボタン（赤） → `/customers/{id}/delete`
  - 「一覧に戻る」ボタン（グレー） → `/customers`

**ユーザー操作**:
- 「編集」ボタンで顧客編集画面へ遷移
- 「削除」ボタンで削除確認画面へ遷移
- 「一覧に戻る」で顧客一覧画面へ戻る

---

### 3.4 顧客登録フォーム（`/customers/new`）

**目的**: 新しい顧客情報を入力して登録する

**主な表示要素**:
- **ページタイトル**: 「新規顧客登録」
- **入力フォーム**（水平レイアウト、3セクション構成）:

**セクション1: 基本情報**

| フィールド     | 入力タイプ | 必須 | プレースホルダー        |
|---------------|-----------|------|----------------------|
| 姓            | text      | ✅   | 山田                  |
| 名            | text      | ✅   | 太郎                  |
| メールアドレス  | email     | ✅   | yamada@example.com    |
| 電話番号       | text      | —    | 090-1234-5678         |
| 住所           | text      | —    | 東京都千代田区...       |

**セクション2: 契約情報**

| フィールド       | 入力タイプ | 必須 | 選択肢 / プレースホルダー                       |
|-----------------|-----------|------|----------------------------------------------|
| 契約番号        | text      | —    | POL-XXXX                                      |
| 契約種別        | select    | —    | 生命保険 / 医療保険 / 自動車保険 / 火災保険        |
| ステータス       | select    | —    | 申請中(0) / 有効(1) / 失効(2) / 解約(3)          |
| 保険料（月額・円）| number    | —    | 5000（step=1）                                 |

**セクション3: 担当者情報**

| フィールド     | 入力タイプ | 必須 | プレースホルダー     |
|---------------|-----------|------|-------------------|
| 担当者名       | text      | —    | 担当者名           |
| 担当者メール   | text      | —    | agent@example.com  |

- **操作ボタン**:
  - 「登録」ボタン（青）
  - 「キャンセル」リンク（グレー） → `/customers`

**ユーザー操作**:
- 必須項目（姓・名・メールアドレス）を入力して「登録」ボタンをクリック
- 登録成功後、顧客詳細画面（`/customers/{id}`）にリダイレクト
- 「キャンセル」で顧客一覧画面に戻る

---

### 3.5 顧客編集フォーム（`/customers/{id}/edit`）

**目的**: 既存の顧客情報を変更・更新する

**主な表示要素**:
- **ページタイトル**: 「顧客編集」
- **入力フォーム**: 顧客登録フォームと同一レイアウト（既存データがプリセット）
- **操作ボタン**:
  - 「更新」ボタン（青）
  - 「キャンセル」リンク（グレー） → `/customers/{id}`

**ユーザー操作**:
- 既存の値を変更して「更新」ボタンをクリック
- 更新成功後、顧客詳細画面（`/customers/{id}`）にリダイレクト
- 「キャンセル」で顧客詳細画面に戻る

> 💡 登録フォームと編集フォームは同じテンプレート（`customers/form.html`）を共有し、`editMode` フラグで表示内容を切り替えています。

---

### 3.6 顧客削除確認（`/customers/{id}/delete`）

**目的**: 顧客削除前に確認を求め、誤操作を防止する

**主な表示要素**:
- **危険パネル**（赤枠）:
  - 「以下の顧客を削除しますか？」タイトル
  - 削除対象の顧客情報テーブル: ID、氏名、メール、契約番号、契約種別
  - **警告メッセージ**: 「この操作は取り消せません。本当に削除しますか？」（赤アラート）
- **操作ボタン**:
  - 「削除する」ボタン（赤） → `/customers/{id}/delete?confirm=yes`
  - 「キャンセル」リンク（グレー） → `/customers/{id}`

**ユーザー操作**:
- 「削除する」をクリックすると顧客が削除され、顧客一覧画面にリダイレクト
- 「キャンセル」で顧客詳細画面に戻る

---

### 3.7 保険料シミュレーション（`/premium-calculator`）

**目的**: 保険種別・年齢・喫煙有無を入力して、月額保険料の見積もりを行う

**主な表示要素**:
- **入力フォームパネル**（左列）:
  - 「計算条件を入力」タイトル

| フィールド | 入力タイプ   | 必須 | 選択肢 / 制約                             |
|-----------|-------------|------|------------------------------------------|
| 保険種別   | select      | ✅   | 生命保険 / 医療保険 / 自動車保険 / 火災保険   |
| 年齢      | number      | ✅   | min=1, max=120, プレースホルダー=30          |
| 喫煙者    | checkbox    | —    | 「はい」ラベル付きチェックボックス             |

  - 「計算する」ボタン（青）

- **計算結果パネル**（右列、計算実行後のみ表示）:
  - 「計算結果」タイトル（緑パネル）
  - **月額保険料**（大きな数字、カンマ区切り、「円/月」単位）
  - 入力条件テーブル: 保険種別（日本語）、年齢（歳）、喫煙者（はい/いいえ）
  - 注意書き: 「※税込金額です。実際の保険料は審査により異なる場合があります。」

**ユーザー操作**:
- 保険種別・年齢を選択/入力し、必要に応じて喫煙者にチェック
- 「計算する」ボタンで保険料を試算（GETリクエスト、同一ページで結果表示）
- 条件を変更して再計算が可能

---

### 3.8 月次レポート（`/report`）

**目的**: 顧客・契約の集計データを月次レポートとして確認する

**主な表示要素**:
- **生成日時**: 「生成日時: YYYY-MM-DD ...」
- **サマリー統計パネル**（5列）:
  - 総顧客数（青パネル）
  - 有効契約数（緑パネル）
  - 失効契約数（赤パネル）
  - 月間保険料合計（水色パネル、カンマ区切り円表示）
  - 平均保険料（黄色パネル、カンマ区切り円表示）
- **全顧客データテーブル**:
  - 列: ID、氏名、契約番号、契約種別（日本語表示）、ステータス（色付きラベル）、保険料（カンマ区切り円表示）、担当者

**ユーザー操作**:
- レポートの内容を確認する（表示のみ、操作なし）
- ナビゲーションバーから他の画面へ遷移

### グローバルナビゲーション

### 画面内ナビゲーション

| 画面           | 遷移先                                        |
|---------------|----------------------------------------------|

### フラッシュメッセージ

以下の操作後に成功メッセージが表示されます:

| 操作       | メッセージ               | 遷移先           |
|-----------|------------------------|-----------------|

| 顧客削除   | 「顧客を削除しました。」   | 顧客一覧画面      |

### レスポンシブ対応

- Bootstrap 3.3.7のグリッドシステムによるレスポンシブデザイン
- モバイル時はナビゲーションがハンバーガーメニューに折りたたみ
- 統計パネルやフォームはモバイル時に縦並びに変更
